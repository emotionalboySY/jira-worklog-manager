# Jira 작업 로그 매니저 — 이용 가이드

Jira의 워크로그(작업 로그)를 효율적으로 기록·관리하기 위한 웹 도구입니다. 브라우저에서 접속해 Jira 계정으로 로그인하면, 본인의 이슈 목록을 보고 작업 시간을 측정·기록·수정할 수 있습니다.

## 로그인 / 로그아웃

### 로그인

1. 첫 방문 시 **"Jira로 로그인"** 버튼이 표시됩니다.
2. 버튼 클릭 → Atlassian OAuth 2.0 로그인 화면으로 이동 → 권한 동의 후 앱으로 돌아옵니다.
3. 로그인 성공 시 사용자 정보 및 접근 가능한 Jira 사이트 정보가 저장되어 API 호출에 사용됩니다.
4. 액세스 토큰이 만료되면(기본 1시간) 자동으로 refresh token을 이용해 갱신됩니다. 갱신이 실패하면 로그아웃 처리됩니다.

### 로그아웃

- 헤더 우측 상단의 **로그아웃** 버튼을 클릭합니다.
- 토큰, 사용자 정보, Cloud ID 등 Jira 관련 localStorage 키가 제거됩니다. 이슈/워크로그 캐시, 즐겨찾기, 세션은 로그아웃으로 삭제되지 않습니다.

## 화면 구성

상단부터 순서대로:

1. **헤더** — 앱 제목, 저장소 안내(ℹ 아이콘), 사용자 이름, 테마 스위치, 로그아웃 버튼
2. **현재 작업 영역** — 진행 중인 작업 세션 카드 + `+ 수동 기록` 버튼
3. **메인 탭 바** — 이슈 목록 / 작업 로그 기록 / 요약
4. **탭 콘텐츠** — 선택된 탭의 내용
5. **즐겨찾기 패널 (우측 고정)** — 별표한 이슈를 빠르게 시작/재개
6. **설정 FAB (좌측 하단)** — 설정 모달 열기

테마 스위치는 좌(라이트) / 우(다크)로 토글됩니다. 테마 설정은 localStorage에 저장됩니다.

## 이슈 목록 탭

내 Jira 이슈를 조회하고 작업을 시작하거나 수동 기록을 남길 수 있습니다.

### 이슈 데이터 소스

- **활성 이슈**: 상태 카테고리가 "Done"이 아닌 모든 이슈 (내가 assignee / reporter / watcher인 것)
- **완료/보류 이슈**: 상태 카테고리 "Done" + 최근 30일 내 업데이트된 것 (토글로 표시/숨김)
- 이슈 목록은 localStorage에 캐시되어 재방문 시 즉시 표시되고, 백그라운드에서 최신 데이터로 업데이트됩니다.

### 검색 바

- **이슈 키 검색 (예: 123, DKT-123)** 입력 필드
  - 숫자만 입력: 모든 프로젝트에서 `프로젝트키-번호` 형태로 매칭 (예: `123` → DKT-123, DK-123 등 모두 조회)
  - `프로젝트키-번호` 형태: 해당 이슈 단건 조회
  - 프로젝트 키만 입력 (예: `DKT`): 해당 프로젝트의 내 이슈 최근 업데이트순 조회
- 500ms 디바운스 자동 검색, Enter로 즉시 검색, Escape로 초기화
- 검색 중에는 필터/프로젝트 탭이 비활성화되며, 결과는 별도 리스트로 표시됩니다.
- **✕** 버튼으로 검색 해제

### 프로젝트 선택 탭

- Jira 프로젝트 목록 중 내 이슈가 있는 프로젝트만 표시됩니다 (`전체` 포함).
- 선택한 프로젝트의 이슈만 필터링해서 보여줍니다.

### 역할 필터 탭

- **전체 / 할당됨 / 보고자 / 워칭** 4가지 필터. 각 탭에 현재 프로젝트 기준 개수 표시.
- 이슈의 "할당/보고/워칭" 태그는 현재 사용자 기준으로 Jira 데이터에서 계산됩니다.

### 옵션 (필터 탭 오른쪽)

- **현재 스프린트만 보기**: 활성(open) 스프린트에 속한 이슈만 표시. 첫 사용 시 스프린트 이슈 키 목록을 조회하느라 잠시 시간이 걸릴 수 있습니다.
- **완료/보류 일감 보기**: 상태 카테고리가 "Done"인 이슈도 리스트에 포함
- **페이지 크기 선택**: 10 / 20 / 30 / 50개씩
- **새로고침 버튼 (↻)**: 이슈 목록 / 스프린트 / 프로젝트를 강제로 다시 조회

### 이슈 행

각 이슈 행에는 다음이 표시됩니다 (좌 → 우):

- **이슈 타입 아이콘 + 라벨** (Jira에서 받아온 이름, 없으면 기본 아이콘)
- **이슈 키** — Jira 페이지로 열리는 링크 (새 탭)
- **요약(제목)**
- **즐겨찾기 별표 토글**
- **상태 배지** (진행중 / 검토 / 완료 등)
- **역할 태그** (할당 / 보고 / 워칭)
- **수동 기록 버튼 (⏱ 아이콘)** — 해당 이슈로 수동 기록 모달 열기
- **시작 / 재개 / 진행 중 버튼** — 작업 세션 제어 (세션이 없으면 "시작", 일시중지 상태면 "재개", 활성 상태면 "진행 중" ↔ 호버 시 "종료" 텍스트로 변함)
- **우클릭 메뉴** — 수동 기록 열기 / 이슈 키·요약·링크 복사

### 이슈 정렬

설정에 따른 상태 순서 > 프로젝트 순서 > 이슈 번호 내림차순으로 정렬됩니다.

### 페이지네이션

- 페이지가 여러 개일 때 1, 2, …, N 표시. 현재 페이지 양쪽 ±2 페이지와 첫/마지막 페이지, 중간은 `...`
- 클릭 시 이슈 리스트 상단으로 자동 스크롤
- 전체 건수도 함께 표시

## 작업 로그 기록 탭

특정 날짜의 작업 로그를 조회하고 수정/삭제할 수 있습니다.

### 툴바

- **새로고침 (↻)** — 현재 월의 작업 로그를 Jira에서 다시 조회
- **달력 열기 / 닫기** — 좌측 달력 뷰의 표시 여부 토글. 상태는 localStorage에 저장되어 새로고침해도 유지됩니다.

### 달력 영역 (열림 상태일 때 좌측)

- 상단 좌우 `◀ ▶` 버튼으로 월 이동. 미래 월로는 이동할 수 없습니다.
- 드롭다운으로 년/월 직접 선택 가능 (미래 월은 disabled).
- **오늘** 버튼 — 오늘 날짜로 이동.
- 날짜 셀은 작업량에 따라 4단계(레벨 0~3)로 색상이 진해집니다.
  - 0 = 없음, 1 = 3시간 미만, 2 = 3~6시간, 3 = 6시간 이상
- 셀 우상단에는 간단한 합계 시간(예: `5시간30분`)이 표시됩니다.
- 연차/반차 설정된 날짜에는 해당 배지가 표시됩니다.
- 오늘/선택된 날짜는 강조 표시.
- **미래 날짜는 클릭할 수 없습니다.**

### 날짜 네비게이션 (달력 위 / 오른쪽 본문 상단)

- 좌우 `◀ ▶` 버튼 (다음 버튼은 오늘 이후로는 비활성)
- 가운데 날짜 입력란 — 클릭 시 한국어 달력 팝업(flatpickr)으로 직접 선택
- 선택한 날짜가 오늘이 아니면 **오늘** 버튼 표시

### 연반차 토글 (로그 상세 상단)

- **없음 / 연차 / 오전 반차 / 오후 반차** 중 하나를 선택
- 선택 값은 브라우저 localStorage에 저장됩니다 (날짜별)
- 요약 탭의 차트에서 해당 날짜에 배지가 표시됩니다.

### 작업 로그 목록

해당 날짜에 Jira에 기록된 나의 워크로그들이 시간 순으로 나열됩니다. 각 로그 행에는:

- **시간대** (HH:MM → HH:MM) — 시작~종료 시간
- **소요 시간**
- **이슈 키 + 요약**
- **점심 배지** — 점심시간 자동 차감이 적용된 경우 `점심 -XX분` 배지 표시
- **코멘트** — 로그에 작성된 코멘트 (있으면)
- **수정 / 삭제 버튼** — 각 로그 조작
- **우클릭 메뉴** — 이슈 수동 기록 / 복사

하단에는 **총 작업 시간**이 집계되어 표시됩니다.

### 데이터 캐싱

- 월별 작업 로그는 최근 3개월까지 localStorage에 캐시됩니다.
- 탭 진입 시 캐시가 있으면 즉시 표시하고, 백그라운드에서 최신 데이터로 검증/업데이트합니다.

## 요약 탭

주간 단위의 작업량을 한눈에 보여줍니다.

### 주차 네비게이션

- 좌우 `◀ ▶` 버튼으로 주 이동 (미래 주는 비활성)
- 가운데 **N월 M주차** 표시 — 설정의 주 시작 요일과 관계없이 **ISO 기준** (해당 주의 목요일이 속한 달)
- **이번 주** 버튼 — 금주로 즉시 이동

### 주 시작 요일

설정 모달에서 **목요일 ~ 수요일** (기본) 또는 **월요일 ~ 일요일** 중 선택할 수 있습니다. 선택한 시작 요일에 따라 주간 차트와 카드의 범위가 달라지지만, 주차 번호는 ISO 식으로 일관됩니다.

### 요약 카드

- **오늘** — 오늘 작업 시간 합계, 기록된 worklog 건수 (금주일 때만 표시)
- **이번 주 / 주간 합계** — 주간 총 작업 시간, 작업 기록이 있는 일수
- **일 평균** — 작업 기록된 날만 기준으로 한 일 평균 시간

### 일별 차트

- 설정한 시작 요일 기준의 주간 7일(기본 목~수, 또는 월~일) 일별 작업 시간을 세로 막대 차트로 표시
- 8시간을 100%로 두고 상대 높이 계산, 넘치는 값도 100%로 클리핑됩니다.
- 주말(토·일)은 스타일로 구분
- 미래 날짜(이번 주 중 아직 안 온 날)는 흐리게 표시
- **막대 클릭 시** 해당 날짜의 작업 로그 기록 탭으로 이동합니다 (달력도 자동으로 열림).
- 연차/반차가 설정된 날에는 배지 표시

## 작업 세션(진행 중 작업) 영역

헤더 바로 아래, 메인 탭 위에 위치합니다. 브라우저 기반 실시간 시간 측정 기능입니다.

### 빈 상태

진행 중 세션이 없으면 안내 문구와 함께 두 버튼이 표시됩니다.

- **+ 수동 기록** — 수동 기록 모달 열기
- **일감 없이 작업 시작하기** — 이슈 지정 없이 바로 세션 시작 (종료할 때 이슈 지정)

### 세션 카드

세션이 있으면 카드가 표시됩니다. 각 카드에는:

- **이슈 키 링크 + 요약** (일감 미지정 세션은 "일감 미지정" 배지)
- **상태 표시** — `● 진행 중` 또는 `⏸ 중단됨`
- **시작 시각 + 구간 수** — 여러 번 일시중지-재개 했으면 `N구간` 표기
- **직전 종료 시간으로** 링크 — 같은 날 기록된 마지막 worklog의 종료 시각으로 이 세션의 시작 시각을 조정 (연속 작업에서 빈 시간이 없도록)
- **타이머** — 활성 중이면 실시간 1초 단위로 증가, 일시중지 중에는 고정 표시
- **중단 / 재개 / 종료 / 취소** 버튼
  - 중단: 세션을 paused 상태로 만들고 현재 구간의 종료 시각 기록
  - 재개: 이전 구간을 닫고 새 구간 추가 (다른 활성 세션이 있으면 자동 중단)
  - 종료: 종료 모달 열기 (Jira에 기록)
  - 취소: 기록 없이 세션 삭제 (취소 확인 모달)

### 동작 규칙

- **한 번에 active 세션은 하나만** 가능합니다. 새 작업을 시작하거나 기존 세션을 재개하면, 기존 active 세션이 자동으로 paused 됩니다.
- **일감 미지정 세션은 최대 한 개만** 가질 수 있습니다 (빈 상태에서만 `일감 없이 작업 시작하기` 버튼이 보임).
- 세션은 브라우저 localStorage에만 저장됩니다. 종료(Jira 기록)하지 않은 채 브라우저 데이터를 삭제하면 사라지므로 주의하세요.
- 카드 하단의 ⚠️ 안내 문구로 이 사실이 고지됩니다.

### 점심 자동 차감

종료 모달에서 Jira 기록 시, 작업 구간이 점심시간(기본 11:30~12:30)과 겹치면 자동으로 그 만큼이 차감되어 두 개의 worklog로 쪼개 기록됩니다 (실제 시작/종료 시각 보존). 점심시간은 모달의 **점심시간** 입력에서 그 기록에만 한해 바꾸거나, **차감 안 함**으로 끌 수 있습니다.

## 즐겨찾기 패널

우측 고정 플로팅 패널.

### 접힘 상태

- 별 모양 버튼 하나. 즐겨찾기한 이슈 수 배지 표시.
- 클릭 시 펼침.

### 펼침 상태

- 상단: `즐겨찾는 이슈` 타이틀, 개수, 접기 버튼(▸)
- 본문: 각 즐겨찾기 이슈 카드
  - 이슈 키 링크 + 요약
  - 상태에 따라 **시작 / 재개 / 진행 중(종료)** 버튼
  - 우측 **✕** 버튼으로 즐겨찾기 해제
  - 우클릭 메뉴 지원

### 즐겨찾기 추가/해제

- 이슈 목록에서 이슈 행의 별표 버튼을 클릭
- 즐겨찾기 패널에서 ✕ 버튼 클릭
- 즐겨찾기 목록은 localStorage에 저장

## 모달 상세 설명

모든 모달은 ESC 키와 각자의 **취소** 버튼으로만 닫힙니다 (오버레이 바깥 클릭은 닫지 않음 — textarea 드래그 선택 중 오탐 방지).

### 종료 모달 (세션의 "종료" 버튼)

- **이슈 정보** — 이슈 키 + 요약 (일감 미지정 세션이면 이슈 키 입력 필드로 대체)
- **작업 구간** — 세션의 각 구간별로 시작/종료 시간을 `HH:MM` 입력으로 수정 가능
  - 마지막(진행 중) 구간에는 **지금** 버튼으로 종료 시간을 현재 시각으로 설정
  - 입력 변경 즉시 구간별 소요 시간과 전체 합계가 실시간 갱신
  - **종료 시간이 시작 시간보다 앞서면** 해당 구간 readout이 빨간색 경고로 바뀝니다
  - 점심 시간이 겹치면 `(점심시간 N분 제외)` 표시
- **실 작업 시간 합계**
- **작업 내용 (코멘트)** — 텍스트에어리어
- **Jira에 기록** 버튼
  - 유효하지 않은 구간이 있으면 alert로 차단
  - 제출 시 버튼이 스피너로 전환, 중복 클릭 차단
  - 성공 시 세션 삭제 + 해당 월 캐시 무효화 + 로그 다시 조회
  - 실패 시 세션은 그대로 유지되고 알림 표시

### 수동 기록 모달 (+ 수동 기록 버튼 / 이슈 행의 ⏱ 버튼 / 컨텍스트 메뉴)

- **이슈 키 입력** — 자동완성 드롭다운 (로컬 이슈 + Jira API 검색 결합)
  - 키보드 ↑↓로 항목 이동, Enter로 선택
  - 입력 후 blur 시점에 대문자 정규화 + 형식·존재 여부 검증
- **작업 날짜** — 오늘까지만 선택 가능 (미래 차단)
- **시작 시간** — 입력 시 해당 날짜의 마지막 worklog 종료 시각으로 자동 채움 (사용자가 직접 수정하면 자동 채움 해제)
- **종료 시간** — `지금` 버튼으로 현재 시각 설정 가능
- **소요 시간** — 실시간 계산, 점심시간 차감 반영
- **작업 내용**
- **Jira에 기록** 버튼 — 점심시간과 겹치면 두 구간으로 쪼개 기록. 실패 시 상세 에러 메시지 표시

### 작업 로그 수정 모달 (로그 행의 "수정" 버튼)

- 이슈 정보(변경 불가), 시작/종료 시간, 소요 시간 readout, 코멘트
- `지금` 버튼으로 종료 시간을 현재 시각으로
- 저장 시 기존 worklog는 첫 구간으로 업데이트, 점심 뒷 구간이 생기면 별도로 신규 생성

### 작업 로그 삭제 모달 (로그 행의 "삭제" 버튼)

- 확인 문구 + `취소 / 삭제`
- 삭제 버튼 클릭 시 스피너로 전환, 중복 클릭 차단
- 실패 시 상세 에러 표시 + 버튼 복구

### 작업 취소 확인 모달 (세션의 "취소" 버튼)

- `아니오 / 취소하기`
- 취소하기 누르면 세션을 삭제 (기록된 시간 소실)

## 설정

좌측 하단의 **⚙ 톱니바퀴 FAB**을 클릭하면 설정 모달이 열립니다. 다시 클릭하면 닫힙니다 (취소와 동일 동작: 미리보기 되돌림).

### 이슈 상태 정렬 순서

- 드래그 핸들로 상태명(예: 진행중 / 검토 / 배포대기 / 준비 / 대기 / 완료됨 / 완료 / 보류 / Closed)을 위·아래로 순서 변경
- 위에 있을수록 이슈 목록 상단

### 프로젝트 정렬 · 색상

- 드래그 핸들로 프로젝트(DK / DKT / DD / RM) 순서 변경
- 각 프로젝트 행의 **색 선택기**로 대표 색상을 변경 가능
  - 변경 즉시 이슈 목록의 컬러 바, 이슈 키 배지 배경/호버 색에 반영 (미리보기)
  - 색상은 bar / fg / bg 세 단계로 자동 파생

### 요약 탭 주 시작 요일

- **목요일 ~ 수요일** (기본) 또는 **월요일 ~ 일요일** 중 선택
- 요약 탭의 주간 범위(차트, 합계, 일평균)가 선택된 시작 요일 기준으로 계산됩니다.
- 주차 번호는 선택과 무관하게 ISO 기준(주의 목요일이 속한 달)으로 일관 유지됩니다.

### 액션 버튼

- **기본값으로 재설정** — 모든 설정을 초기값으로 되돌림 (즉시 저장)
- **취소** — 변경 사항 폐기 후 닫기
- **저장** — 현재 설정을 영구 저장 (localStorage)

## 점심시간 자동 차감

- 기본 점심시간: **11:30 ~ 12:30** (설정 모달 > **기본 점심시간**에서 변경 가능)
- 수동 기록 / 세션 종료 / 작업 로그 수정 시, 작업 시간이 이 구간과 겹치면 해당 분수만큼 자동으로 차감됩니다.
- 내부적으로 하나의 긴 작업 구간을 점심 앞·뒤 두 개의 worklog로 쪼개서 저장하므로, Jira 상에서 원래 시작·종료 시각이 그대로 보존됩니다.
  - 예) 10:03~12:37 작업 → Jira에 `10:03~11:30` + `12:30~12:37` 두 개로 기록, 실 작업 1시간 34분

### 그날만 점심시간 바꾸기 / 차감 끄기

- 수동 기록·세션 종료·작업 로그 수정 모달에는 **점심시간** 입력(시작 ~ 종료)이 있고, 기본값은 설정의 기본 점심시간으로 채워집니다.
- 특정 날만 점심시간이 다르면(예: 행사·경기 시청으로 10:00~12:00) 그 모달에서만 값을 바꿔 기록하면 됩니다. 설정의 기본값은 그대로 유지됩니다.
- 점심을 거른 날 등 차감이 필요 없으면 **차감 안 함**에 체크하면 그 기록은 점심을 빼지 않고 통째로 저장됩니다.
- ⚠️ 데스크톱 위젯은 아직 기본 점심시간(11:30~12:30) 고정입니다. 위젯에서 종료할 때 예외 처리가 필요하면 웹앱에서 기록하세요.

## 데이터 저장 위치 / 주의사항

이 앱은 **브라우저 localStorage에만** 다음 데이터를 저장합니다. 서버 DB가 없습니다.

| 키                                                                    | 내용                     |
| -------------------------------------------------------------------- | ---------------------- |
| `jira_access_token` / `jira_refresh_token` / `jira_token_expires_at` | Jira OAuth 토큰          |
| `jira_cloud_id` / `jira_site_name` / `jira_user`                     | Jira 사이트 및 사용자 정보      |
| `work_sessions`                                                      | 진행 중/일시중지 세션 (구간 포함)   |
| `day_offs`                                                           | 날짜별 연차/반차 설정           |
| `favorite_issues`                                                    | 즐겨찾기한 이슈 목록            |
| `issues_cache`                                                       | 이슈 목록 캐시               |
| `worklog_cache`                                                      | 월별 작업 로그 캐시 (최근 3개월)   |
| `user_preferences`                                                   | 설정 모달의 정렬 순서 / 프로젝트 색상 |
| `theme`                                                              | 다크 / 라이트 테마            |
| `favorites_collapsed` / `log_calendar_open`                          | 패널·달력 접힘 상태            |

### 주의

- **Jira에 기록하지 않은 진행 중 세션은 브라우저에만 있습니다.** 브라우저 캐시 · 사이트 데이터 삭제, 시크릿 모드, "Clear Cache"류 확장 사용 시 사라질 수 있습니다.
- 즐겨찾기, 연차/반차 설정, 프로젝트 색상 등도 브라우저 localStorage 기반이므로 다른 기기/브라우저에서는 공유되지 않습니다.
- 이슈/워크로그는 캐시가 지워져도 Jira에서 다시 받아올 수 있으므로 안전하지만, 세션만은 반드시 종료(=Jira에 기록)한 뒤 브라우저 데이터를 건드리세요.

## 홍보용 스크린샷 자동 촬영

프로젝트 루트에서 아래 명령을 실행하면 실제 Jira 계정에 로그인하지 않고 홍보용 데모 화면을 자동 촬영합니다.

```bash
npm run screenshot
```

촬영 결과는 `screenshots/` 폴더에 다음 파일로 생성됩니다.

- `01-이슈-목록.png`
- `02-작업-로그.png`
- `03-주간-요약.png`

촬영 과정은 격리된 브라우저에서 가상 사용자·이슈·작업 기록만 사용합니다. 실제 Jira 토큰, 사용자 정보, 워크로그를 읽거나 수정하지 않습니다. Chrome을 찾지 못한다는 안내가 나오면 `npx playwright install chromium`을 한 번 실행한 뒤 다시 촬영하세요.

## FAQ

**Q. 작업 구간이 여러 개인데 구간마다 Jira 워크로그가 여러 개 생성됩니다.**
A. 의도된 동작입니다. 일시중지·재개를 반복한 세션은 각 구간별로 worklog를 남기고, 점심시간과 겹친 구간도 앞·뒤 두 개로 쪼개 기록합니다. 이렇게 해야 Jira 상에서 실제 작업 시간대가 그대로 보존됩니다.

**Q. 수동 기록에서 시작 시간이 자동으로 채워졌어요.**
A. 해당 날짜에 이미 기록된 worklog 중 가장 늦은 종료 시각을 기본 시작 시간으로 넣어줍니다. 연속 작업 시 공백 없이 이어 기록할 수 있게 하기 위함입니다. 직접 수정하면 자동 채움이 꺼집니다.

**Q. "직전 종료 시간으로" 링크를 누르면 어떻게 되나요?**
A. 현재 세션의 **첫 구간 시작 시각**을 같은 날 기록된 worklog 중 가장 늦은 종료 시각으로 조정합니다. 앞선 작업과 공백 없이 이어 기록하고 싶을 때 유용합니다. 조정 후 시각이 현재 시작 시각보다 늦으면 경고가 표시됩니다.

**Q. 이슈 키를 입력했는데 "이슈를 찾을 수 없습니다" 경고가 뜹니다.**
A. 이슈 키 형식(영문 대문자로 시작하는 프로젝트 키 + 숫자, 예: `DKT-123`)이 맞는지 확인하세요. 존재하더라도 내 계정이 접근 권한이 없으면 조회되지 않을 수 있습니다.

**Q. 달력 레벨(색상 진하기)은 어떻게 결정되나요?**
A. 해당 날짜의 작업 로그 총합을 기준으로 0 = 기록 없음, 1 = 3시간 미만, 2 = 3~6시간, 3 = 6시간 이상입니다.
